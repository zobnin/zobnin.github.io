Вместо введения

Как я уже сказал, о Tasker написано огромное количество статей и уроков, поэтому я лишь кратко остановлюсь на том, что это такое и сразу перейду к главному.

По своей сути Tasker - инструмент автоматизации смартфона. Он позволяет назначать запуск определенных действий в ответ на события, происходящие в системе. С помощью Tasker, например, можно сделать так, чтобы в ответ на подключение к смартфону наушников (событие) был запущен аудио-плеер (действие) или при наступлении 20:00 (событие) яркость экрана была снижена (действие).

Tasker поддерживает огромное количество действий и событий и все их можно использовать для составления своих правил автоматизации. Правила называются "профилями" и состоят из указанного события и набора привязанных к нему действий, которые образуют "задачу". Далее в статье я расскажу о нескольких профилях, которые использую сам, возможно среди них вы найдете полезные для себя.

Выключение экрана взмахом руки

Чтобы выключить экран достаточно потрясти смартфоном влево-вправо. Вместо выключения можно использовать любое другое действие, поддерживаемое Tasker, например, включение фонарика. 

Контекст: событие -> sensor -> shake -> Axis: Left-Right
Задача: экран -> блокировать

Автоматическое включение "режима полета" по ночам

Чтобы сэкономить заряд батареи и защитить себя от "звонков не вовремя" можно использовать автоматическое включение режима полета ночью. В данных настройках режим полета будет включен с 1 часу ночи до 7 утра. Для работы профиля требуется плагин Secure Settings, который можно найти в Play Store (требует root).

Контекст: время -> с 1:00 до 7:00
Задача: плагин -> Secure Settings -> Root Actions -> Airplane Mode

Запуск аудио-плеера при подключении наушников

Здесь все просто: воткнул наушники - запустился плеер.

Контекст: состояние -> аппаратура -> наушники подключены
Задача: приложение -> запустить приложение -> выбираем нужный плеер

Управление яркостью в зависимости от времени

Профиль пригодится тем, кто использует устройства без датчика освещения. С 9 утра до 8 вечера яркость будет выставлена на 1/3 от максимальной (значение 100, макс. - 255), в остальное время - на минимум.

Контекст: время -> с 9:00 до 20:00
Задача: экран -> яркость дисплея -> 100
Контекст: время -> с 20:01 до 8:59
Задача: экран -> яркость дисплея -> 0

Включение раздачи интернета при подключении по USB

При подключении смартфона к ПК будет автоматически включена функция USB-тизеринга

Контекст: состояние -> подключено по USB
Задача: сеть -> интернет по USB

Включение энергосберегающего режима при падении уровня заряда батареи до 30%

Если уровень заряда батареи упадет до 30 процентов, яркость экрана будет снижена до минимальной, отключен wifi и GPS, режим сети принудительно переведен на 2G. 

Контекст: состояние -> уровень зарядки -> от 0 до 30
Задача:
    экран -> яркость дисплея -> 0
    сеть -> моб. данные -> только 2G
    сеть -> статус Wifi -> выключить
    плагин -> Secure Settings -> System+ Actions -> GPS -> Off

Активация "домашнего режима" при подключении к указанной wifi-сети

Данный профиль основан на идее о том, что смартфон должен функционировать немного по-разному в зависимости от того, находится ли его владелец дома или за его пределами. Так, например, громкость звонка дома лучше увеличить, так как смартфон может находится в другой комнате, синхронизация с сервисами Google в домашних условиях обычно не нужна, так как почту и сообщения можно проверить на большом ПК, GPS и PIN на экране блокировке дома также бесполезны, но функция ADB over Wifi может пригодится для быстрого подключения к смартфону с ПК.

Контекст: состояние -> сеть -> подключено к WiFi-сети -> имя сети
Задача:
    аудио -> громкость звонка -> 7
    сеть -> авто-синхронизация -> выключить
    плагин -> Secure Settings -> System+ Actions -> GPS -> Off
    плагин -> Secure Settings -> Root Actions -> Wireless ADB -> On
    плагин -> Secure Settings -> Dev Admin Actions -> Password/Pin -> Disabled

Потребуется также "обратный" профиль, который будет активироваться при выходе из дома (отключении от указанной сети)

Контекст: состояние -> сеть -> подключено к WiFi-сети -> "если не"
Задача:
    аудио -> громкость звонка -> 4
    сеть -> авто-синхронизация -> включить
    плагин -> Secure Settings -> System+ Actions -> GPS -> On
    плагин -> Secure Settings -> Root Actions -> Wireless ADB -> Off
    плагин -> Secure Settings -> Dev Admin Actions -> Password/Pin -> Enabled

Анти-вор своими руками

Это простейший анти-вор. Его задача: отправить информацию о местоположении устройства (а также уровне заряда батареи) в ответ на СМС-сообщение "locate", полученное с указанного номера. 

Контекст: телефон -> Получено текст. сообщение -> Тип: SMS, Отправитель: "+7...", Содержание: "locate"
Задача:
    плагин -> Secure Settings -> System+ Actions -> GPS -> On
    task -> ждать -> 30 секунд
    разное -> получить координаты -> Источник: любой
    телефон -> отправить СМС -> Номер: "+7...", Сообщение: "Date: %DATE %TIME. Battery: %BATT. Location: %LOC"
